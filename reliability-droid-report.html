Alert Investigation – AWS Amplify shows "Welcome" placeholder for EduMyles-ERP-SaaS

SEV-2 Started: Unknown (reported by user)
On-call: Myles Corp Ltd

Executive Summary

AWS Amplify Hosting displays the default "Welcome – Your app will appear here once you complete your first deployment" page for the EduMyles app. This occurs when the build succeeds but Amplify does not find an index.html in the configured artifacts baseDirectory or the build output directory points to the wrong path in a monorepo. The repository is a Turborepo monorepo with the Next.js app at apps/web; no amplify.yml is present, and the app’s Next.js config outputs standalone (SSR) by default. Amplify is likely configured for Static Hosting with an incorrect baseDirectory or build commands that never generate an index.html in the location Amplify serves from.

Root Cause Analysis

Suspected root cause:

  • Primary Hypothesis: Amplify build settings point to the wrong artifact path for a monorepo Next.js app (apps/web), so the final artifact directory is empty or missing index.html; Amplify therefore shows the placeholder page.
  • Contributing Factors: No amplify.yml in repo; Next.js config uses output: 'standalone' (SSR) which does not produce an out/index.html unless next export or output: 'export' is used for Static Hosting.

Proposed Fix

  1. Immediate Mitigation (choose one hosting model):

    Option A — Static Hosting (requires purely static pages):

    amplify.yml
version: 1
applications:
  - appRoot: .
    frontend:
      phases:
        preBuild:
          commands:
            - corepack enable
            - corepack prepare pnpm@8.15.0 --activate
            - pnpm install --frozen-lockfile
        build:
          commands:
            - cd apps/web
            - pnpm build
            - npx next export -o out
      artifacts:
        baseDirectory: apps/web/out
        files:
          - '**/*'
      cache:
        paths:
          - node_modules/**/*
          - apps/web/.next/cache/**/*

    Add output: 'export' to apps/web/next.config.js if required for SSG-only export.

    Option B — Amplify SSR for Next.js (recommended for SSR/app router):

    amplify.yml
version: 1
applications:
  - appRoot: .
    frontend:
      phases:
        preBuild:
          commands:
            - corepack enable
            - corepack prepare pnpm@8.15.0 --activate
            - pnpm install --frozen-lockfile
        build:
          commands:
            - cd apps/web
            - pnpm build
      artifacts:
        baseDirectory: apps/web/.next
        files:
          - '**/*'
      cache:
        paths:
          - node_modules/**/*
          - apps/web/node_modules/**/*
          - apps/web/.next/cache/**/*

    Ensure the Amplify app is created as "Amplify Hosting with SSR (Next.js)". This uses the .next output and serves SSR correctly.

    Validation: Trigger a new Amplify build; verify the artifact directory contains index.html (Static) or the full .next bundle (SSR). The live URL should no longer show the placeholder.
  2. Short-term Fix: In Amplify Console → App settings → Build settings, paste the chosen amplify.yml and set the correct baseDirectory (apps/web/out for Static, apps/web/.next for SSR). Confirm the build runs from monorepo root and installs dependencies with pnpm via Corepack.
    Validation: The build logs should show commands executing in apps/web and successful artifact upload.
  3. Long-term Solution: Commit amplify.yml to the repo root and standardize monorepo deployment docs. Add CI checks to validate that the artifact directory exists after build. Consider environment-specific scripts for SSR vs Static.
